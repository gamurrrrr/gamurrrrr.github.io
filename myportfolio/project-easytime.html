The Business Problem

While working in retail, I witnessed managers struggling with static spreadsheets to manage staff. This manual process led to communication errors, missed shifts, and inability to calculate payroll costs in real-time.

The Solution

I built EasyTime to move this workflow from static Excel sheets to a dynamic database. This centralizes scheduling, reduces admin time by ~40%, and eliminates payroll calculation errors.

Project Management & Strategy

1. Defining Scope & Timeline

Using a Gantt Chart, I broke the project down into 6 distinct milestones, from "Project Charter Acceptance" to "Final Product Signoff." This allowed us to identify critical path dependencies, such as ensuring the Backend API was fully tested before Front-end integration began.

Figure 1: Work Breakdown Structure (WBS) & Timeline

2. Risk Management & Pivots

Mid-project, we faced a critical resource risk: a key backend developer was unavailable, putting 47% of tasks at risk.

Strategic Decision: Scope Reduction

To save the release deadline, I formally proposed a Change in Scope. We moved "Exporting Features" (CSV/PDF) to low priority to focus resources on the core backend functionality.

Technical Implementation

Role-Based Access

Implemented distinct views for "Employer" vs "Employee," restricting sensitive payroll data to authorized accounts only.

Data Integrity

Built a MySQL database to handle complex relationships between Users, Shifts, and Pay Rates.
